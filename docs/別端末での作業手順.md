# 別端末での作業手順

このプロジェクト（シアレ株式会社サイト）を、別のPCで作業するための手順です。
**このページは GitHub 上でそのまま読めます**（clone不要）。別端末のブラウザで開いて、上から順に進めてください。

リポジトリ: https://github.com/atsushi20101010-cyber/siare （非公開）

---

## A. その端末で「初回だけ」やること

### ① 必要ツールを入れる（無ければ）
```powershell
winget install --id Git.Git          # Git
winget install --id GitHub.cli       # GitHub CLI（gh）
```
インストール後はターミナルを開き直すとPATHが反映されます。

### ② GitHubにログイン
```powershell
gh auth login
```
プロンプトは次のように選択:
1. **Where do you use GitHub?** → `GitHub.com`
2. **Preferred protocol?** → `HTTPS`
3. **Authenticate Git with your GitHub credentials?** → `Yes`
4. **How would you like to authenticate?** → `Login with a web browser`
5. 表示の**ワンタイムコード**をメモ → Enter → ブラウザで貼り付けて承認

`✓ Logged in as atsushi20101010-cyber` が出ればOK。

### ③ コミット名義を設定（任意・最初の1回）
```powershell
git config --global user.name "あなたの名前"
git config --global user.email "GitHubのメール"
```

### ④ プロジェクトを取得（clone）
OneDriveとの二重同期を避けるため、**OneDrive外の場所**（例: `C:\dev\`）にcloneするのがおすすめです。
```powershell
cd C:\dev
git clone https://github.com/atsushi20101010-cyber/siare.git
cd siare
```

---

## B. 「毎回」の作業の流れ（基本サイクル）

```powershell
cd siare          # プロジェクトへ移動

git pull          # ① 作業前：必ず最新を取得

#   …編集・Claude Codeでの作業…

git add -A                    # ② 変更をまとめる
git commit -m "変更メモ"      # ③ コミット
git push                      # ④ 作業後：必ず共有
```

合言葉：**「始める前に pull、終わったら push」**

---

## C. プレビュー（表示確認）
ブラウザで `index.html` を直接開けば表示できます。
サーバ経由で見たい場合（Perlがあれば）:
```powershell
perl _server.pl 4173
# → http://127.0.0.1:4173
```
Perlが無ければ任意の静的サーバでも代用可:
```powershell
python -m http.server 4173
```

---

## ⚠️ 競合を防ぐポイント
- **同じファイルを2台で同時に編集しない**（片方ずつ）
- 作業の**前に必ず `git pull`**、**後に必ず `git push`**
- push時に「rejected（拒否）」が出たら他端末の変更が先にある合図。次で解消:
  ```powershell
  git pull
  git push
  ```
  競合表示が出た場合は、その内容をClaudeに貼れば解消を手伝ってもらえます。
